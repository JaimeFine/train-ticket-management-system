# 火车票务管理系统 — 核心流程图

## 1. 登录与身份验证流程

```mermaid
flowchart TD
    A([用户打开系统]) --> B[显示登录界面]
    B --> C{选择入口}
    C -->|账号登录| D[输入用户名和密码]
    C -->|游客访问| E[直接生成游客身份<br/>role=Guest]
    D --> F{输入是否为空?}
    F -->|是| G[提示: 用户名和密码不能为空]
    G --> B
    F -->|否| H[查询数据库 User 表]
    H --> I{用户名是否存在?}
    I -->|否| J[提示: 用户名或密码错误]
    J --> B
    I -->|是| K{密码是否正确?}
    K -->|否| J
    K -->|是| L{账户是否启用?<br/>enabled=1?}
    L -->|否| M[提示: 账号已被禁用]
    M --> B
    L -->|是| N{判断角色 role}
    N -->|role=2| O[进入管理员界面<br/>可管理账号和车次]
    N -->|role=1| P[进入售票员界面<br/>可办理票务]
    N -->|role=0| Q[进入游客界面<br/>仅查询]
    E --> Q
    O --> R([登录完成])
    P --> R
    Q --> R
```

## 2. 管理员 — 用户新增流程

```mermaid
flowchart TD
    A([管理员进入账号管理]) --> B[点击新增用户]
    B --> C[输入用户名/密码/角色/启用状态]
    C --> D{输入合法性检查}
    D -->|用户名为空| E[提示: 用户名不能为空]
    D -->|密码为空| F[提示: 密码不能为空]
    D -->|角色值非法| G[提示: 角色值必须为 0/1/2]
    E --> C
    F --> C
    G --> C
    D -->|通过| H[查询用户名是否已存在]
    H --> I{用户名重复?}
    I -->|是| J[提示: 用户名已存在]
    J --> C
    I -->|否| K[执行 INSERT INTO User]
    K --> L{插入成功?}
    L -->|否| M[提示: 数据库写入失败]
    M --> C
    L -->|是| N[提示: 创建成功]
    N --> O[刷新用户列表]
    O --> P([完成])
```

## 3. 车次新增流程

```mermaid
flowchart TD
    A([管理员进入车次管理]) --> B[点击新增车次]
    B --> C[输入车次号/出发站/到达站<br/>时间/座位数/启用状态]
    C --> D{车次号是否重复?}
    D -->|是| E[提示: 车次号已存在]
    E --> C
    D -->|否| F{出发站是否存在于 Station 表?}
    F -->|否| G[提示: 出发站不存在]
    G --> C
    F -->|是| H{到达站是否存在于 Station 表?}
    H -->|否| I[提示: 到达站不存在]
    I --> C
    H -->|是| J{出发站与到达站是否相同?}
    J -->|是| K[提示: 出发站和到达站不能相同]
    K --> C
    J -->|否| L{座位数合法性检查}
    L -->|totalSeats ≤ 0| M[提示: 总座位数必须大于 0]
    L -->|remainingSeats > totalSeats| N[提示: 余票数不能超过总座位数]
    L -->|remainingSeats < 0| O[提示: 余票数不能为负数]
    M --> C
    N --> C
    O --> C
    L -->|通过| P[INSERT INTO Train]
    P --> Q{写入成功?}
    Q -->|否| R[提示: 数据库写入失败]
    R --> C
    Q -->|是| S[提示: 车次创建成功]
    S --> T([完成])
```

## 4. 车次查询与修改流程

```mermaid
flowchart TD
    A([进入车次管理]) --> B{查询方式}
    B -->|按车次号| C[输入车次号]
    B -->|按车次ID| D[输入车次ID]
    C --> E[SELECT FROM Train]
    D --> E
    E --> F{查询结果}
    F -->|不存在| G[提示: 未找到该车次]
    F -->|存在| H[展示车次详情]
    H --> I{是否修改?}
    I -->|否| J([结束])
    I -->|是| K[修改车次信息]
    K --> L{修改后合法性检查}
    L -->|车次号与他人重复| M[提示: 车次号冲突]
    L -->|违反数据约束| N[提示: 数据不合法]
    L -->|通过| O[UPDATE Train]
    M --> K
    N --> K
    O --> P{更新成功?}
    P -->|否| Q[提示: 更新失败, 保留原数据]
    Q --> H
    P -->|是| R[提示: 修改成功]
    R --> J
```

## 5. 订单创建流程（订票）⭐ 核心

```mermaid
flowchart TD
    A([用户进入售票页签]) --> B[输入搜索条件<br/>出发站/到达站/日期]
    B --> C[searchTrains<br/>JOIN Train + Station]
    C --> D{搜索结果}
    D -->|无结果| E[提示: 未找到符合条件的车次]
    E --> B
    D -->|有结果| F[表格展示车次列表<br/>余票颜色标注]
    F --> G[用户选择车次<br/>输入乘客姓名]
    G --> H{乘客姓名非空?}
    H -->|否| I[提示: 请输入乘客姓名]
    I --> G
    H -->|是| J[bookTicket 开始]
    J --> K{车次是否存在?}
    K -->|否| L[提示: 车次不存在]
    L --> G
    K -->|是| M{车次是否启用?<br/>enabled=1?}
    M -->|否| N[提示: 该车次已停运]
    N --> G
    M -->|是| O{余票 > 0?}
    O -->|否| P[提示: 该车次已无余票]
    P --> G
    O -->|是| Q[adjustTrainSeats<br/>remainingSeats - 1]
    Q --> R{扣座成功?}
    R -->|否| S[提示: 扣减座位失败]
    S --> G
    R -->|是| T[createOrder<br/>INSERT INTO Order<br/>status=0 Booked]
    T --> U{写入成功?}
    U -->|否| V[回滚: adjustTrainSeats +1]
    V --> W[提示: 创建订单失败]
    W --> G
    U -->|是| X[弹窗显示订单详情<br/>订单号/乘客/车次/站名]
    X --> Y[刷新车次列表]
    Y --> Z([完成])
```

## 6. 退票流程

```mermaid
flowchart TD
    A([进入退改签页签]) --> B[输入乘客姓名或订单号]
    B --> C[findOrderById 或<br/>findOrdersByPassenger]
    C --> D{订单是否存在?}
    D -->|否| E[提示: 订单不存在]
    E --> B
    D -->|是| F[展示订单列表<br/>状态颜色标注]
    F --> G[用户选择订单]
    G --> H{订单状态 = Booked?}
    H -->|否| I[提示: 该订单无法退票<br/>已退票或已改签]
    I --> G
    H -->|是| J[弹出确认弹窗]
    J --> K{用户确认?}
    K -->|否| G
    K -->|是| L[updateOrderStatus<br/>status = 1 Refunded]
    L --> M[adjustTrainSeats<br/>remainingSeats + 1]
    M --> N[提示: 退票成功]
    N --> O[刷新订单列表]
    O --> P([完成])
```

## 7. 改签流程

```mermaid
flowchart TD
    A([进入退改签页签]) --> B[查询并选择订单<br/>状态 = Booked]
    B --> C[输入新车次号]
    C --> D{新车次是否存在?}
    D -->|否| E[提示: 找不到该车次号]
    E --> C
    D -->|是| F{新车次是否启用?}
    F -->|否| G[提示: 目标车次已停运]
    G --> C
    F -->|是| H{新车次余票 > 0?}
    H -->|否| I[提示: 目标车次已无余票]
    I --> C
    H -->|是| J[弹出确认弹窗]
    J --> K{用户确认?}
    K -->|否| B
    K -->|是| L[旧订单: updateOrderStatus<br/>status = 2 Changed]
    L --> M[旧车次: adjustTrainSeats +1]
    M --> N[新车次: adjustTrainSeats -1]
    N --> O{扣座成功?}
    O -->|否| P[回滚全部操作]
    P --> Q[提示: 改签失败]
    Q --> B
    O -->|是| R[createOrder 新订单<br/>status = 0 Booked]
    R --> S{写入成功?}
    S -->|否| P
    S -->|是| T[提示: 改签成功]
    T --> U[刷新订单列表]
    U --> V([完成])
```

## 8. 订单查询与历史流程

```mermaid
flowchart TD
    A([进入订单查询]) --> B{查询方式}
    B -->|按订单号| C[findOrderById]
    B -->|按乘客姓名| D[findOrdersByPassenger<br/>LIKE 模糊匹配]
    B -->|按用户ID| E[findOrdersByUser]
    B -->|全部订单| F[findAllOrdersWithDetails<br/>JOIN Train + Station]
    C --> G{结果}
    D --> G
    E --> G
    F --> G
    G -->|无结果| H[提示: 未找到订单]
    G -->|有结果| I[表格展示<br/>订单号/车次/站名/乘客/时间/状态]
    I --> J{状态颜色}
    J -->|Booked| K[绿色: 已预订]
    J -->|Refunded| L[红色: 已退票]
    J -->|Changed| M[蓝色: 已改签]
    H --> N([结束])
    K --> N
    L --> N
    M --> N
```

## 9. 统计信息流程

```mermaid
flowchart TD
    A([进入统计页签]) --> B[自动触发 onRefreshStatistics]
    B --> C[countAllOrders<br/>countOrdersByStatus × 3]
    C --> D[更新4个概览卡片<br/>总订单/已预订/已退票/已改签]
    B --> E[findAllOrdersWithDetails]
    E --> F[填充订单历史表格]
    B --> G[popularRoutes<br/>GROUP BY 路线 LIMIT 10]
    G --> H[填充热门路线表格<br/>Top3 奖牌标注]
    B --> I[monthlyPassengerFlow<br/>SUBSTR 按月聚合]
    I --> J[填充月度客流表格]
    D --> K([完成])
    F --> K
    H --> K
    J --> K
```

## 10. 保存退出流程

```mermaid
flowchart TD
    A([用户点击退出]) --> B[关闭当前功能界面]
    B --> C{有待保存数据?}
    C -->|是| D[SQLite 自动落盘<br/>数据已持久化]
    C -->|否| E[无需额外保存]
    D --> F[closeDatabase<br/>关闭数据库连接]
    E --> F
    F --> G[QApplication::quit]
    G --> H([程序正常退出])
```

## 数据库 ER 图

```mermaid
erDiagram
    User ||--o{ Order : "userId"
    Train ||--o{ Order : "trainId"
    Station ||--o{ Train : "departureStationId"
    Station ||--o{ Train : "arrivalStationId"

    User {
        int userId PK
        string username UK
        string password
        int role "0=Guest 1=Seller 2=Admin"
        int enabled "default 1"
    }

    Station {
        int stationId PK
        string stationName UK
    }

    Train {
        int trainId PK
        string trainNumber UK
        int departureStationId FK
        int arrivalStationId FK
        string departureTime
        string arrivalTime
        int totalSeats
        int remainingSeats
        int enabled "default 1"
    }

    Order {
        int orderId PK
        int userId FK
        int trainId FK
        string passengerName
        string purchaseTime
        int status "0=Booked 1=Refunded 2=Changed"
    }
```
